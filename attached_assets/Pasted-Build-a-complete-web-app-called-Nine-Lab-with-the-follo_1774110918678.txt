=== APP OVERVIEW ===
Nine Lab is a placement preparation tool for Indian students.
Student pastes their Resume + Job Description + Company Name,
and gets 3 downloadable PDFs in return.

- Backend: Python FastAPI with uvicorn

- Search: Tavily API (tavily-python library)
- PDF: ReportLab library
- Database: Supabase (supabase library)
- Frontend: HTML + CSS + Vanilla JS (no React)
- All async/await throughout

STAGE 1 - Run BOTH in parallel using asyncio.gather():

  agent_research(company_name, jd):
    Use Tavily API - run these 3 searches:
    Search 1: "{company_name} interview process rounds India 2024"
    Search 2: "{company_name} company overview revenue funding"
    Search 3: "{company_name} salary freshers India {job_title}"
    
    Then use Gemini to summarize into:
    - company_overview (what they do)
    - why_hiring (reason from JD context)
    - interview_rounds (list of rounds)
    - salary_range (India specific)
    - company_culture (values, work style)
    - financial_health (revenue, growth, funding)
    
    Return: company_data dict
    
    If Tavily fails: use only Gemini with company name
    to generate best possible estimates.
    Add note: "Data JD se generate kiya gaya"

  agent_analysis(resume, jd):
    Use Gemini API with this exact prompt structure:
    - Extract all skills from resume as list
    - Extract all requirements from JD as list  
    - Find matching skills (in both)
    - Find missing skills (in JD but not resume)
    - Calculate match_score out of 10
    - Identify candidate's strongest points
    - Identify biggest weaknesses for this role
    
    Return: analysis dict

STAGE 2 - Merge:
  master_context = {
    company_data: from agent_research,
    analysis: from agent_analysis,
    original_resume: resume,
    original_jd: jd,
    company_name: company_name
  }

STAGE 3 - Run BOTH in parallel using asyncio.gather():

  agent_plan(master_context):
    Use Gemini API:
    - Look at missing skills from analysis
    - Create priority topics list (most important first)
    - For each topic:
      * Why important for this company
      * Best YouTube search query
      * Best book/resource
      * Time needed (hours)
    - Create day-wise 7-day study plan
    - Interview tips specific to this company
    - Common questions for this role
    
    Return: prep_plan dict

  agent_resume(master_context):
    Use Gemini API:
    - Extract top 15 keywords from JD
    - Rewrite resume summary/objective for this role
    - Suggest where to add keywords naturally
    - Improve bullet points with action verbs
    - Add quantification suggestions
    - Format for ATS compatibility
    - Keep candidate's real experience intact
    
    Return: new_resume dict
